# ftrace: трассировка системных вызовов

Лабораторная работа: простая программа на C, настройка **ftrace**, сбор данных
о системных вызовах и анализ результатов.

## Состав

| Файл | Назначение |
|------|------------|
| `syscall_demo.c` | Программа на C, выполняющая 11 различных системных вызовов |
| `Makefile` | Статическая сборка программы |
| `trace.sh` | Настройка ftrace и сбор трасс (события `syscalls` + `function_graph`) |
| `analyze.sh` | Частотный анализ перехваченных системных вызовов |
| `run_demo.sh` | Полный прогон «под ключ» в выделенной Linux‑VM (см. ниже) |
| `results/` | Реальные собранные данные (трассы и их анализ) |
| `REPORT.md` | Подробный отчёт с анализом результатов |

## Быстрый запуск на любом Linux с ftrace

```sh
make                       # собрать ./syscall_demo
sudo ./trace.sh            # настроить ftrace и собрать трассы в results/
./analyze.sh results/10_syscalls.trace
```

Требования к ядру: `CONFIG_FTRACE=y`, `CONFIG_FTRACE_SYSCALLS=y`,
`CONFIG_FUNCTION_GRAPH_TRACER=y` (есть в стоковых ядрах Ubuntu/Fedora) и права
root.

## Как это запускалось в данной среде

Работа выполнялась на macOS (Apple Silicon), где у хоста нет Linux‑ядра с
ftrace. Поэтому для получения **настоящих** данных ftrace был поднят отдельный
одноразовый Linux‑VM (Fedora CoreOS, ядро 6.19, ftrace включён) через
`podman machine`. Скрипт `run_demo.sh` автоматизирует: создание VM → сборку
программы в контейнере → запуск `trace.sh` внутри VM → выгрузку результатов →
удаление VM. Детали и обоснование — в `REPORT.md`, раздел «Среда выполнения».
